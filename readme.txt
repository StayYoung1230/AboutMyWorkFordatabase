Steam 價格小助手

--------------------------------------------------------------------------------------
檔案說明：

main.py：將暢銷頁面的前 n 個遊戲抓取並存入資料庫。

app.py：Flask 後端，用來處理使用者請求並顯示查詢結果。

templates/index.html：網頁主體，顯示查詢界面與結果。

checkDB.py：用來查看資料庫內已抓取的資料（偵錯用途）。

static/styles.css：網頁樣式表（CSS），定義 index.html 的外觀樣式。
--------------------------------------------------------------------------------------

程式流程說明：

首先，我們使用 main.py 程式來抓取 Steam 暢銷頁面的前 n 款遊戲資料，並將這些資料儲存在名為 mydb4final.db 的 SQLite 資料庫中。資料內容包括遊戲名稱、開發商、發行商、標籤、區域及價格等。

接著，利用 app.py 搭建的 Flask 後端，提供查詢功能，讓使用者可以根據遊戲名稱、標籤以及價格範圍來查詢遊戲。查詢結果會顯示最低價格的遊戲資訊以及對應的區域。

查詢結果會即時展示在 templates/index.html 頁面上，並透過 static/styles.css 調整頁面的外觀樣式，提供簡潔易用的查詢界面。
--------------------------------------------------------------------------------------

程式補充說明：

資料量控制：在 main.py 中，您可以透過調整 max_pages 來控制抓取資料的頁數。max_pages 代表從 Steam 商店的暢銷頁面抓取的頁數，您可以設定為任何正整數。每一頁預設抓取 25 款遊戲。如果您希望抓取更多遊戲，可以根據需求調整每頁抓取的數量。

控制抓取頁數的方式：修改 main.py 中的 max_pages 變數，將其設為您需要抓取的頁數。例如，將 max_pages = 5 會抓取前 5 頁的遊戲資料。

查詢功能：您可以在網頁的查詢框內輸入遊戲名稱、標籤（Tag）、或是設定價格範圍，系統會顯示符合條件的遊戲結果，並顯示遊戲的最低價格以及對應的區域。

標籤篩選：支持多個標籤的查詢，您可以使用「,」分隔多個標籤來進行篩選。

資料庫結構：資料庫包括以下表格：

Game：儲存遊戲名稱及相關資訊。

Developer：儲存遊戲開發商資料。

Publisher：儲存遊戲發行商資料。

Tag：儲存遊戲的標籤資訊。

PriceRecord：儲存每個區域的遊戲價格。

GameTag：關聯遊戲與標籤的多對多關係。
--------------------------------------------------------------------------------------

如何執行：

安裝必要的套件

首先，確保您的系統已經安裝了 Python 3.x。

安裝必要的 Python 套件：

pip install flask sqlite3


執行 main.py 抓取資料：

打開 main.py，在裡面設定 max_pages 變數，指定您希望抓取的頁數。

執行 main.py 來抓取 Steam 資料並存入資料庫：

python main.py


完成後，您會在同一目錄下看到 mydb4final.db 資料庫檔案。

啟動 Flask 後端：

執行 app.py 启动 Flask 伺服器：

python app.py
--------------------------------------------------------------------------------------

開啟網頁：

打開瀏覽器，進入 http://127.0.0.1:5000/，您就可以看到查詢系統的結果。
